Comments

Function: Add some notes and explanations to the code to make it easier for yourself or other programmers programmers to read the code

Two formats

Single line comments: // descriptive information
usually placed above a line of code, or at the end of a statement, == description of the line of code ==
Multi-line comments: /_ descriptive information _/
Usually placed at the top of a line of code, == an overall description of the line of code ==
Tip: The compiler ignores comments when compiling code

Identifier naming rules

Function: C++ has its own rules for naming identifiers (variables, constants)

Identifiers cannot be keywords.
Identifiers can only consist of letters, numbers, and underscores
The first character must be a letter or an underscore
Letters in identifiers are case-sensitive
